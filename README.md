# ayx-ssl-archive

本仓库用于归档和发布一组独立的 HTML 页面。这些页面不针对任何特定域名或网站，仅作为静态内容存档使用。

## 项目简介

ayx-ssl-archive 是一个轻量的静态页面归档仓库。其目的是将多个独立的 HTML 文件集中存放，便于版本管理、历史追溯与网络分发。仓库本身不提供动态服务，也不包含任何服务端逻辑。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── index.html      # 可选的入口页面（如有）
└── README.md       # 本说明文件
```

- **pages/**：主目录，每个 HTML 文件代表一个独立归档页面。
- **assets/**：页面所需的公共或私有资源文件，按需组织子目录。
- **index.html**：如存在，可作为浏览时的起始页。

## 页面归档说明

- 所有页面均为纯静态 HTML，不依赖后端或数据库。
- 页面内容已剥离具体域名、网址及营销信息，仅保留通用结构与展示内容。
- 归档文件以时间或编号命名，方便按序查阅。
- 若页面引用了外部资源（如字体库、图标库），建议一并归档至 assets/ 目录，确保离线可用。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在必要时更新 `assets/` 中的资源。
- 删除页面：直接移除对应文件，建议在提交信息中注明原因。
- 版本管理：遵循语义化提交记录，每次归档变更应有清晰描述。
- 兼容性：页面应尽量适配主流浏览器，避免使用已废弃的 HTML 或 JavaScript 特性。

本仓库仅用于技术归档与学习交流。页面内容版权归原作者所有，如有需要请参考各页面内的注释或声明。
